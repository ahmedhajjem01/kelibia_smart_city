2. reclamations/models.py Modifié

Quoi et pourquoi ?

Ajout d'un champ GPS (PointField) au modèle Reclamation pour stocker la position géographique de chaque signalement.

Champ ajouté :

from django.contrib.gis.db import models as gis_models

class Reclamation(models.Model):
    # ... champs existants ...
    location = gis_models.PointField(
        geography=True,
        null=True,
        blank=True,
        srid=4326           # Système WGS84 (GPS standard, utilisé par Google Maps)
    )
SRID 4326 : C'est le système de coordonnées GPS mondial (latitude/longitude). C'est le même que Google Maps, OpenStreetMap, Leaflet.js utilisent.

4. reclamations/serializers.py Modifié

Quoi et pourquoi ?

Le serializer convertit les objets Python (modèles Django) en JSON pour l'API REST. On a ajouté le support du champ location en format GeoJSON.

from rest_framework_gis.serializers import GeoFeatureModelSerializer

class ReclamationSerializer(GeoFeatureModelSerializer):
    class Meta:
        model = Reclamation
        geo_field = "location"       # Champ géographique principal
        fields = [
            'id', 'title', 'description', 'category',
            'status', 'location', 'created_at', 'citizen'
        ]
GeoFeatureModelSerializer retourne les données au format GeoJSON — standard utilisé par Leaflet.js pour afficher des marqueurs sur une carte.

5. reclamations/views.py Modifié

Quoi et pourquoi ?

Les vues exposent les endpoints API. On a ajouté un endpoint pour récupérer les réclamations avec leurs coordonnées GPS pour la carte interactive.

from rest_framework.permissions import IsAuthenticated
from rest_framework_gis.filters import InBBoxFilter

class ReclamationViewSet(viewsets.ModelViewSet):
    queryset = Reclamation.objects.all()
    serializer_class = ReclamationSerializer
    permission_classes = [IsAuthenticated]
    bbox_filter_field = 'location'        # Filtre par zone géographique
    filter_backends = (InBBoxFilter,)

    def get_queryset(self):
        # Un citoyen ne voit que ses propres réclamations
        # Un agent voit toutes les réclamations
        user = self.request.user
        if user.user_type == 'agent' or user.is_staff:
            return Reclamation.objects.all()
        return Reclamation.objects.filter(citizen=user)

// Appel à l'API Django pour obtenir un token JWT
const response = await fetch('http://127.0.0.1:8000/api/token/', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ email, password })
});

const data = await response.json();

// Stockage des tokens dans localStorage
localStorage.setItem('accessToken', data.access);
localStorage.setItem('refreshToken', data.refresh);

// Redirection selon le rôle de l'utilisateur
if (data.is_staff || data.is_superuser) {
    window.location.href = 'http://127.0.0.1:8000/admin/';
} else if (data.user_type === 'agent') {
    window.location.href = 'agent_dashboard.html';
} else {
    window.location.href = 'dashboard.html';
}

9. run_frontend.py Modifié

Quoi et pourquoi ?

Serveur HTTP simple pour servir les fichiers HTML/JS/CSS du frontend. Le port a été changé de 5500 à 5501 car 5500 était déjà occupé (VS Code Live Server).

import http.server, socketserver, os

PORT = 5501                              # Changé de 5500 → 5501
FRONTEND_DIR = os.path.join(
    os.path.dirname(__file__), 'frontend'
)

os.chdir(FRONTEND_DIR)
with socketserver.TCPServer(("", PORT), http.server.SimpleHTTPRequestHandler) as httpd:
    print(f"Serving Frontend at http://127.0.0.1:{PORT}")
    httpd.serve_forever()
